# 步學吾數 視覺設計規格 v2（十人設計團隊產出，2026-07-16 確認執行）

## 1. 吉祥物「數字精靈」
- 造型：2.5 頭身 Q 版、大眼睛、簡易四肢；半透明霓虹玻璃體材質（該主幹符號構成半透明色塊 + 同色系霓虹光暈描邊）
- 5 主幹變體：數與量＝分數線精靈（青綠 #4fd1c5）、代數＝x/y 符號精靈（暖橙 #f6ad55）、空間與形狀＝幾何塊精靈（紫藍 #7c9cff，未上線）、關係與規律＝函數曲線精靈（黃綠 #a3e635，未上線）、資料與可能性＝圖表精靈（珊瑚粉 #f687b3，未上線）
- 5 種狀態：idle（技能樹站樁）／happy 答對／sad 答錯鼓勵（非負面）／celebrate 精熟慶祝／guide 引導提示

## 2. 色彩字體版面系統
- 正確/錯誤一律「色＋符號」雙重編碼（✓／✕），不單靠色相
- 5 主幹識別色相間隔 ≥60°
- 卡片圓角 12px→16px，陰影改彩色光暈
- 字體：標題/節點名可換偏圓潤字重，數字獨立等寬字體
- 完整 CSS 變數表見 `css/style.css` `:root`

## 3. 技能樹：路徑地圖化
- SVG 節點圖取代 grid 卡片，prereq 用貝茲曲線連接（鎖定虛線、解鎖實線）
- 精靈站在「目前最前緣可挑戰節點」上方
- 精熟度：環形進度條，滿 80% 後轉星等（1-3★）
- 徽章解鎖：全螢幕慶祝卡（精靈慶祝姿態＋遊戲化文案）
- 「即將推出」主幹：半透明剪影地圖＋霧化精靈剪影＋輕互動提示氣泡

## 4. 測驗畫面與動效
- 4 題型視覺差異化（概念辨識＝左右大按鈕、錯誤診斷＝病歷卡風格、情境應用＝情境小圖示）
- 答對/答錯：純 CSS 動效（光暈脈動/搖晃）+ 精靈對應表情動畫
- 連對 ≥3：題卡持續發光＋連擊數字
- 進度：分段進度條取代純文字
- 結束總結：三段式過關儀式（精靈慶祝→星等逐顆點亮→戰報卡片）

## 5. 可用性與無障礙
- 標記「建議下一步」節點收斂視覺焦點
- 手機響應式斷點＋44px 最小觸控熱區
- 首次使用：非阻斷式教學貼紙（sessionStorage 記錄，不用 alert/prompt）
- 鎖定節點顯示前置條件文字，而非只有「未解鎖」
- 文案更遊戲化：精熟度→戰力值、錯題本→錯題復仇本、已解鎖→已開通
