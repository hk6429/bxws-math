# 步學吾數四模組升級計畫書（2026-07-20）

由字字珠璣四大模組改版計畫改寫而來，錨定步學吾數既有架構。
前置基礎（已上線）：boss 戰引擎、收集品傷害加成（cap 15%）、PvP 本機挑戰書（mulberry32 seed）。
既有資產：110 節點（num-quantity 61／algebra 5／space-shape 22／relation-pattern 9／data-uncertainty 13）、
星屑系統（`getStardustCount`＋里程碑，目前只進不出）、數學家精靈立繪（davinci／gauss…）、175 項測試全綠。

## 總覽

| Phase | 模組 | Task 數 | 後端 | 預估工期 |
|---|---|---|---|---|
| A | 星靈融合 fusion | 11 | 不需要 | 1 天 |

施工順序 A→B→C→D。A、B 純前端先上線；C 後端只部署 Cloudflare 線（Pages Functions＋D1），
Vercel／Netlify 自動偵測無 API 退回本機模式，三平台皆不壞。

## 全案設計不變式

1. 一切數值掛真實學習：星屑只從「首次答對／首通試煉／精熟節點」產出，不從點擊產出
2. 戰鬥加成總上限 25%（收集 15%＋星靈 10%），刷收集永遠贏不過答對率
3. 無隨機開箱、明碼標價——市集不能有賭博感
4. 白帽排行：只露前五，永不公開墊底
5. 雙親不消耗——融合素材永不損失，零挫折懲罰
6. 不動 `mastery-engine.js` 與收集掉落規則；每 phase 維持既有測試全綠

## Phase A 星靈融合（11 tasks）

核心設計：融合表就是乘法表，配方規則就是數學本身。

- 每隻星靈＝一個數字（2～100）。融合：`a × b → a×b`（乘積 ≤100）
- 質數星靈無法由融合誕生，只能奇遇捕獲——疑問本身在教質數
- 平方數融合（a×a）產出閃光「平方星靈」；完全數 6、28 為隱藏傳說靈
- 星屑經濟：沿用 `getStardustCount`，新增 `stardustSpent` 消費帳本，餘額＝總獲得−總花費

| # | Task | 產出 |
|---|---|---|
| A1 | `js/fusion.js` 純邏輯層：因數分解／質數判定／融合合法性／星靈圖鑑資料 | pure functions＋單元測試 |
| A2 | 星屑錢包：`stardustSpent` 帳本、餘額計算、store key 註冊 | 測試鎖死「只進學習事件」 |
